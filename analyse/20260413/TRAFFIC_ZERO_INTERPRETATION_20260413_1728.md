# 교통량(Vol) 0 해석 메모 (2026-04-13 17:28)

## 1) 교통량 0인데 0이 아닌 값이 반영된 도로가 있는가?
- 기준 노선 3개(`1050000571`, `1020003121`, `1070000091`)는 재호출 XML 기준 `vol`이 전건 0입니다.
- 이 3개 노선의 평균 교통량(`mean_vol`)도 모두 0입니다.
- 반면 비교 노선 `1010001001`(수도권 제1순환고속도로)는 같은 시점에 `vol` 0/비0가 혼재합니다.
- 결론: "모든 도로가 0인데 계산에서 0이 아닌 값이 생긴다"는 패턴은 확인되지 않았고, 0이 아닌 값은 실제로 원천 `vol` 비0가 존재하는 구간에서 발생합니다.

## 2) 실제 교통량이 0이 아닐 확률은?
- 통계적으로는 "알 수 없음"이 정확합니다. 현재 데이터만으로 확률을 엄밀 추정할 수 없습니다.
- 다만 정성적으로는 "실제 0이 아닐 가능성"이 있습니다.
- 근거:
	- 동일 시점 같은 API에서 다른 노선은 `vol` 비0를 정상 제공함
	- 문제 노선도 `spd`, `trvlTime`은 유효값이 존재함
	- 즉, 측정 채널 중 `vol`만 누락/0 처리되었을 가능성을 배제하기 어렵습니다.

## 3) 원천 API 이상 가능성은?
- 가능성 있음.
- 단, API 자체 장애(HTTP 실패)보다는 "필드 단위 데이터 품질/매핑 이슈" 가능성이 더 큽니다.
- 이유:
	- 경기 OpenAPI는 호출 자체는 정상(HTTP 200, `headerCd=0`)
	- 특정 routeId에서만 `vol` 전건 0이 반복 관측됨
	- 동일 API의 다른 routeId에서는 `vol` 비0가 제공됨

## 4) 운영 권고
- 보고/분석 시 `vol`을 확정치로 단정하지 말고 "해당 노선은 vol 관측 불완전" 주석을 유지하세요.
- 혼동 방지를 위해 `vol_observed_nonzero_ratio = nonzero_count / vol_rows`를 지표로 함께 제시하세요.
- 필요 시 보조지표(`spd`, `trvlTime`) 기반의 별도 혼잡도 지표를 병행하세요.
