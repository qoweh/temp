# 도로 현황2.png 출처 추적 + 상승 결빙도로(그라데이션) 제작 가이드

## 1. 도로 현황2.png는 어떤 파일들로 만들어졌나

`도로 현황2.png`는 QGIS에서 드래그 앤 드롭으로 만든 결과라고 하셨으므로, 실제 레이어 출처는 아래 도로 파이프라인 산출물로 보는 것이 맞다.

### 1-1) 생성 체인(원천 -> 중간 -> QGIS 레이어)

1. 도로 형상(선) 생성
- 스크립트: `plus/build_uiwang_osm_road_inventory.py`
- 주요 산출물: `plus/uiwang_osm_roads_filtered.geojson`

2. 도로별 교통속성 결합(CSV)
- 스크립트: `plus/build_uiwang_road_traffic_merge.py`
- 주요 산출물: `plus/uiwang_all_roads_with_traffic.csv`

3. 선형 도로 + 교통속성 결합(GeoJSON 라인)
- 스크립트: `plus/build_uiwang_qgis_line_layer.py`
- 주요 산출물: `plus/uiwang_all_roads_with_traffic_lines.geojson`

4. 경계 기준/검증 반영 라인 파일
- 관련 검증 스크립트: `plus/validate_uiwang_roads_boundary.py`
- 경계 파일: `second report/gpt/this_uiwang_boundary.geojson`
- 실사용 라인: `plus/uiwang_all_roads_with_traffic_lines_in_boundary.geojson`

5. 교통량 추정(inferred) 보강 라인
- 스크립트: `plus/build_uiwang_road_traffic_infer.py`
- 산출물: `plus/uiwang_all_roads_with_traffic_lines_in_boundary_inferred.geojson`

### 1-2) QGIS에 바로 넣기 좋은 최종 파일(도로 관련)

실무용으로는 아래 파일 1개면 충분하다.

- `second report/결과물/qgis_final/uiwang_roads_traffic_inferred.geojson`

참고: `second report/결과물/qgis_final/README.md`에도 최종 실사용 파일로 명시되어 있다.

## 2. 상습 결빙도로 시각화.png처럼 그라데이션으로 만드는 방법

이번에 QGIS Heatmap용 입력 파일을 새로 생성했다.

- 생성 스크립트: `plus/build_uiwang_icing_heatmap_points.py`
- 생성 결과(활용 파일): `plus/uiwang_road_icing_heatmap_points.geojson`
- 생성 요약: `plus/uiwang_road_icing_heatmap_points_summary.json`

이 파일은 도로 선분을 점으로 분해하고, `kde_weight`(가중치)를 넣어 Heatmap(KDE)에 바로 쓸 수 있게 만든 파일이다.

## 3. QGIS 작업 순서 (재현 절차)

1. 레이어 로드
- `plus/uiwang_road_icing_heatmap_points.geojson` 로드
- (선택) 배경 도로 확인용으로 `second report/결과물/qgis_final/uiwang_roads_traffic_inferred.geojson` 같이 로드
- (선택) 경계 마스크용으로 `second report/gpt/this_uiwang_boundary.geojson` 로드

2. Heatmap 생성
- 레이어 스타일 -> 단일심볼 대신 `Heatmap` 렌더러 선택
- Weight from field: `kde_weight`
- Radius: 먼저 250~400m에서 시작 (예: 300m)
- Color ramp: `Blues` 또는 흰색->진청색 램프
- Render quality를 높이고, Opacity는 70~90% 범위에서 조정

3. 이미지처럼 보이게 다듬기
- 경계 폴리곤(`this_uiwang_boundary.geojson`)이 있으면
  - 경계 채우기: 연회색
  - 경계선: 흰색 얇게
- Heatmap 레이어를 경계 위에 올리고 블렌딩을 `Multiply` 또는 `Normal + 투명도`로 조정
- 레이블은 동 이름(내손1동/청계동 등) 표시

4. 범례/문구 맞추기
- 레이아웃 관리자에서 제목 문구 추가
  - 예: `※ 색이 진할수록 결빙 빈도가 높음`
- 범례 항목 이름을 `상습 결빙 구간 도로`로 변경

5. PNG 내보내기
- Project -> New Print Layout
- 지도/범례/제목 배치
- Export as PNG

## 4. 추천 파라미터(샘플)

- Heatmap Radius: 300m
- Max value: 자동(초기), 필요 시 수동으로 상위 95%에 맞춤
- Weight field: `kde_weight`
- Color ramp stops 예시:
  - 0.00 = #f7fbff
  - 0.35 = #c6dbef
  - 0.60 = #6baed6
  - 1.00 = #08306b

## 5. 체크포인트

- 핫스팟이 너무 퍼지면: Radius를 줄인다(300 -> 220m)
- 점처럼 끊겨 보이면: Radius를 키운다(300 -> 400m)
- 특정 구간만 과하게 진하면: `kde_weight` 대신 `log(1+vol_mean)` 기반 가중치로 바꿔 재시도
