# 의왕시 교통단속 CCTV 추가 설치 분석 요구사항 (Agent 실행용)

## 1) 목표 (Objective)

- QGIS 기반 분석을 위한 데이터 전처리/시각화 자료 생성
- 최종 목적: **의왕시 교통단속 CCTV 추가 설치 후보 위치 도출**

---

## 2) 입력 데이터 범위 (Input Scope)

원칙: `자료/` 디렉토리의 파일을 **전체 검토**하되, 최종 의사결정은 교통안전 관련 객관 지표 중심으로 수행한다.

### 핵심 입력

1. `자료/2024년 시군구별 교통사고.xlsx`
     - 필터: 시도 == `경기`, 시군구 == `의왕시`
     - 사용: 사고(건), 사망(명), 부상(명)

2. `자료/CCTV현황(제공표준).csv`
     - 사용: 위도, 경도, 설치목적구분
     - 핵심 필터: 설치목적구분 == `교통단속`

3. `자료/경기도_교통사고집계현황.csv`
     - 사용: 집계년도, 발생건수(건), 사망자수(명), 부상자수(명)

4. `자료/사망교통사고현황.csv`
     - 사용: WGS84위도, WGS84경도

5. `자료/버스정류소현황.xls`
     - 필터: 시군명 == `의왕시`
     - 사용: 정류소명, 정류소id, WGS84위도, WGS84경도

### 보조 입력 (교차검증/근거 보강)

- `자료/사고다발지현황.csv`
- `자료/demo_의왕_사고다발지.geojson`
- `자료/kosis_의왕_추출결과/*`
- `자료/경기도 의왕시_보행약자 이용시설 현황(보행안전지수)_20221202/*` (좌표/속성 확인 후 활용 가능 항목만 반영)

## 3) 처리 로직 (Processing Logic)

## Step 1. 데이터 정제

- 좌표(위도/경도) 결측 제거
- 좌표계는 WGS84 기준으로 통일
- 컬럼명/형식 통일(숫자형 변환, 단위/공백 정리)

## Step 2. 교통단속 CCTV 추출

- `설치목적구분 == 교통단속`만 남김
- 산출물: `traffic_cctv_uiwang.csv`

## Step 3. 사고 위험지역 도출 (다중 근거)

- 사망교통사고 위치 기반 밀집 분석(Heatmap/KDE)
- 사고다발지 CSV/GeoJSON 자료와 교차 검토
- 가능하면 다중 기준으로 핫스팟 생성 (예: KDE 상위 구간 + 기존 사고다발지 레이어 중첩)
- 산출물: `accident_hotspots.csv`

## Step 4. CCTV 사각지대 분석 (다중 시나리오)

- CCTV 커버리지 버퍼를 단일값이 아닌 **복수 시나리오**로 분석
    - 기본 시나리오 포함: 150m
    - 추가 시나리오 포함: 250m (및 필요 시 추가값)
- 추가로, 실제 CCTV 간 최소거리(또는 분포)를 계산해 거리 기준 근거를 제시하고 시나리오에 반영
- 사각지대 정의:
    - 사고 위험지역 ∩ 교통단속 CCTV 미커버
    - 버스정류장 주변 ∩ 교통단속 CCTV 미커버

## Step 5. 버스정류장 위험 분석 (다중 시나리오)

- 버스정류장 버퍼도 복수 반경 시나리오 분석
    - 기본 시나리오 포함: 50m
    - 추가 시나리오 포함: 150m (및 필요 시 추가값)
- 정류장별 CCTV 존재 여부 플래그 생성
- 산출물: `bus_stop_uiwang.csv`

## Step 6. 최종 후보지 도출

- 후보 조건:
    - (사고 위험지역 포함 OR 버스정류장 근접)
    - AND 교통단속 CCTV 미커버
- 유형 분류:
    - `사고다발`
    - `정류소`
    - `복합`
- 위험도 점수는 가능하면 산정(근거 컬럼 포함)
- 산출물: `cctv_candidate_locations.csv`

## 4) 출력물 정의 (Outputs)

## A. CSV (QGIS 로딩용)

1. `traffic_cctv_uiwang.csv`
2. `accident_hotspots.csv`
3. `bus_stop_uiwang.csv`
4. `cctv_candidate_locations.csv`

공통 규칙:

- 위도/경도 컬럼 필수
- WGS84 기준 유지
- QGIS에서 바로 불러올 수 있는 단순 테이블 구조

## B. Jupyter Notebook

- 파일명: `notebooks/analysis.ipynb` (신규)
- 구성 원칙: **하나의 셀 = 하나의 시각화**

필수 셀 구성:

1. 의왕시 교통사고 통계(막대그래프)
2. 교통단속 CCTV 위치 시각화
3. 사망사고 밀집(heatmap)
4. 버스정류장 + 교통단속 CCTV 오버레이
5. 최종 설치 후보지 시각화

시각화 작성 규칙:

- 각 셀에 제목 포함
- 폰트/스타일 수정이 쉬운 단순 코드 구조 유지

## 5) 분석 기준 / 제약조건 (Critical Rules)

- 핵심 분석 대상 CCTV는 `교통단속`만 사용

- 모든 판단은 데이터 근거 기반으로 수행
