# API 주소 -> 생성 파일 정리 (2026-04-13 17:28 기준 재검증)

## 1) API -> 산출 파일 매핑
- `https://openapigits.gg.go.kr/api/rest/getRoadInfoList` -> `gg_road_info_list.csv`
- `https://openapigits.gg.go.kr/api/rest/getRoadTrafficInfoList` -> `gg_road_traffic_info_list.csv`
- `https://overpass-api.de/api/interpreter` -> `osm_uiwang_bbox_highways.json` (원시) -> OSM 후처리 산출물
- OSM + 경기교통 결합 -> `uiwang_gg_route_traffic_stats.csv`, `uiwang_gg_routeno_traffic_stats.csv`, `uiwang_named_roads_with_traffic_summary.csv`, `uiwang_all_roads_with_traffic.csv`, `uiwang_traffic_data_coverage.json`

## 2) 재호출 결과 요약
- 경기 OpenAPI 두 엔드포인트: 정상 응답(HTTP 200, headerCd=0)
- Overpass: 재호출 시점에서 504 Gateway Timeout 관측
- road.re: `/itms`는 TLS 검증 on에서 인증서 오류, 검증 off 시 응답/리다이렉트 확인, 실제 search 엔드포인트는 200
- VWorld: 본 환경에서 Empty reply(HTTP 000)
- 한국도로공사 trafficAll: 본 환경에서 timeout(HTTP 000)

## 3) 교통량 0 관련 핵심
- `안양판교로/관악대로(1050000571)`, `봉담과천로(1020003121)`, `덕영대로/오봉로(1070000091)`는 재호출 시 vol이 전건 0
- 같은 시점에도 `수도권 제1순환고속도로(1010001001)`는 vol이 0/비0 혼재
- 따라서 평균 0은 계산 로직 문제보다 원천 API vol 필드 값 특성으로 보는 것이 타당
